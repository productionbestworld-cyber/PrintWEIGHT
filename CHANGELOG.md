## v1.0.0 — 2026-06-08  (เวอร์ชันเริ่มต้นที่เริ่มจดบันทึก)

ฟีเจอร์ที่มีในเวอร์ชันนี้ (สรุปรวม):

**หน้าชั่งน้ำหนัก**
- ชั่งม้วน 3 ประเภท: ม้วนดี / ม้วนกรอ / เศษเสีย (ใส/สี/ก้อน)
- ต่อเครื่องชั่งผ่าน Bridge + ระบบ offline queue (เน็ตหลุดชั่งต่อได้)
- เลือก Item Code → เติมข้อมูลอัตโนมัติ (สินค้า/ขนาด/ลูกค้า/Mat Code/แกน)
- พิมพ์ Mat Code → เด้งข้อมูลสินค้า (ค้นย้อนกลับ)
- Item Code ใหม่ → กดเพิ่มเป็นสินค้าใหม่ได้ทันที
- ระบบ "จำ" Mat Code/แกน ที่กรอกเอง (back-fill เข้า master)
- แถบสรุป: ยอดม้วนดี + ยอดดี+กรอ รวม (ทั้งหน้าชั่ง + การ์ดเลือกเครื่อง)

**แผนกกรอ**
- แบนเนอร์แจ้งม้วนรอกรอ + badge นับอัตโนมัติ
- รับม้วน → เริ่มกรอ / ส่งคืนผลิต / เศษ

**คลังข้อมูล** (พนักงานเข้าได้ ไม่ต้อง PIN)
- ลูกค้า 261 ราย + สินค้า 639 รายการ (ตาราง + Export Excel)
- นำเข้า/เพิ่ม/แก้ไข สินค้า (มี Mat Code + น้ำหนักแกน)

**Dashboard** — ศูนย์ควบคุม / ภาพรวม Yield / ปัญหา&สาเหตุ / เปรียบเทียบ / รายงาน + Export ทุกหน้า

**ผู้จัดการ** (ล็อก PIN) — พิจารณาม้วนกรอ / NC / Admin (ใบปะหน้า/เปลี่ยน PIN/Backup ทั้งระบบ)

**อื่นๆ**
- ใบปะหน้า: ใบยาว / ใบสั้น / ใบปะเศษ (ออกแบบเองได้ + ทดสอบปริ้น)
- ล็อกหน้าผู้จัดการด้วย PIN
- โลโก้ลูกโลก BWP (favicon + ในแอป)
- ตัวติดตั้งแอป + Bridge (คลิกเดียว, ASCII รองรับทุกเครื่อง)
- Bridge ค้นหา COM port อัตโนมัติ (Plug & Play)

## วิธีจดเวอร์ชันใหม่ (สำหรับครั้งต่อไป)
1. แก้เลขใน `src/version.ts` (เช่น 1.0.0 → 1.0.1)
2. เพิ่มหัวข้อใหม่บนสุดของไฟล์นี้ บอกว่าแก้อะไร
3. commit แล้วสร้าง tag:  `git tag -a v1.0.1 -m "รายละเอียด"`  →  `git push origin v1.0.1`

## 🔙 วิธีย้อนกลับเวอร์ชันเดิม (Rollback)

**ทางที่ 1 — Vercel (เร็วสุด ไม่ต้องแก้โค้ด):**
- เข้า Vercel → โปรเจค → แท็บ Deployments
- หา deployment เวอร์ชันที่ต้องการ → กด ⋯ → **Promote to Production** (หรือ Rollback)
- เว็บกลับไปเวอร์ชันนั้นทันที

**ทางที่ 2 — Git tag (ย้อนโค้ดจริง):**
```
git checkout v1.0.0      # ดู/ย้อนไปเวอร์ชันนั้น
git revert <commit>      # ถอยเฉพาะการเปลี่ยนแปลงที่ไม่ต้องการ
```

> 💡 เลขเวอร์ชัน: X.Y.Z — Z=แก้บั๊คเล็ก / Y=ฟีเจอร์ใหม่ / X=เปลี่ยนใหญ่
