## v1.1.0 — 2026-07-14
- แยกจอชั่งเป็น "ชั่งพิมพ์" กับ "ชั่งสลิท" คนละหน้า — เลือกเครื่องสลิทได้อิสระตอนชั่งสลิท (ไม่ต้องกำหนดตั้งแต่ตั้งงาน) แต่พองานถูกกำหนดเครื่องสลิทแล้วจะล็อกเครื่องนั้นตลอดจนจบงาน (เปลี่ยนได้เฉพาะ admin)
- Lot ปรับเป็น 2 จังหวะ: ตั้งงานได้ "lot เครื่องพิมพ์" ก่อน (แสดง `__` แทนเครื่องสลิทที่ยังไม่กำหนด) แล้วเติมเป็น lot ครบตอนเลือกเครื่องสลิท
- ม้วนใหญ่หลังพิมพ์ชั่งได้โดยไม่บังคับผูกม้วนต้นทางอีกต่อไป (ผูกได้ถ้าต้องการสอบย้อนกลับ)
- ตั้งงาน: เลือก "ชื่อบริษัทบนใบปะหน้า" ได้จากดรอปดาวน์ (บริษัทหลัก + อีก 2 บริษัทในเครือ) — ใช้กับงานที่ต้องพิมพ์ในนามบริษัทอื่น
- แก้เลขม้วน: เปิดเติมเลขที่ว่าง (gap-fill) ทุกแผนก — ลบม้วนที่ชั่งผิดแล้วม้วนถัดไปได้เลขเดิมกลับมา
- แก้บั๊กม้วนต้นทาง/ยอดสรุป "ม้วนพิมพ์" ขึ้น 0.00 ที่จอชั่งสลิท
- เพิ่มหน้า "Log ชั่ง" — บันทึกการชั่งจริงทุกม้วนทุกสเตจ พร้อมอ้างอิงม้วนต้นทาง (ม้วนสลิทมาจากม้วนพิมพ์ม้วนไหน เวลาอะไร WO/SO ไหน) ค้นหา/กรองช่วงวันที่/Export Excel

## v1.0.0 — 2026-07-09
เวอร์ชันเริ่มต้นแผนกพิมพ์ — แยกโปรเจกต์และฐานข้อมูลออกจากระบบชั่งม้วนเดิม

- 6 หน้าหลัก: หน้าแรก · ตั้งงาน · ชั่ง · โอน · ม้วนพักไว้ · ตั้งค่า
- ตั้งงาน: เจน Lot อัตโนมัติ = ปี + เครื่องสลิท + เครื่องพิมพ์ + รหัสสินค้า + เดือน (เช่น `69SL1P1TC00107`)
- ชั่งแยกตามขั้น: ม้วนก่อนพิมพ์ / ม้วนใหญ่หลังพิมพ์ / ม้วนสลิทส่งลูกค้า + เศษ (สี/กาว/ข้างสลิท) พร้อมสรุปยอดแต่ละขั้นอัตโนมัติ
- ม้วนพักไว้ (เมตรไม่ถึง / แก้ไข / NCR): เก็บพักไว้แล้วเบิกเข้างานเพื่อทำต่อได้
- ชั่งม้วนใหญ่ได้โดยไม่บังคับผูกม้วนต้นทาง (ผูกได้ถ้าต้องการสอบย้อนกลับ)
- ใบปะหน้า 76×76 (ออกแบบเองได้), ล็อกหน้าตั้งค่าด้วย PIN

## วิธีจดเวอร์ชันใหม่ (สำหรับครั้งต่อไป)
1. แก้เลขใน `src/version.ts` (เช่น 1.0.0 → 1.0.1)
2. เพิ่มหัวข้อใหม่บนสุดของไฟล์นี้ บอกว่าแก้อะไร
3. commit แล้วสร้าง tag:  `git tag -a v1.0.1 -m "รายละเอียด"`  →  `git push origin v1.0.1`

## 🔙 วิธีย้อนกลับเวอร์ชันเดิม (Rollback)

**ทางที่ 1 — Vercel (เร็วสุด ไม่ต้องแก้โค้ด):**
- เข้า Vercel → โปรเจค → แท็บ Deployments
- หา deployment เวอร์ชันที่ต้องการ → กด ⋯ → **Promote to Production** (หรือ Rollback)
- เว็บกลับไปเวอร์ชันนั้นทันที

**ทางที่ 2 — Git tag (ย้อนโค้ดจริง):**
```
git checkout v1.0.0      # ดู/ย้อนไปเวอร์ชันนั้น
git revert <commit>      # ถอยเฉพาะการเปลี่ยนแปลงที่ไม่ต้องการ
```

> 💡 เลขเวอร์ชัน: X.Y.Z — Z=แก้บั๊คเล็ก / Y=ฟีเจอร์ใหม่ / X=เปลี่ยนใหญ่
